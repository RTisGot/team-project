\# Comment Rules



\## 基本方針



* コメントは「なぜ必要か」を書く
* コードを見れば分かる内容は書かない
* 可読性向上を目的とする



\---



\# コメントを書く場所



\## 複雑な処理



以下は必ずコメントを書く:



* 当たり判定
* 状態遷移
* 数学処理
* シェーダ制御
* AI処理
* 一時対応



\---



\## 処理ブロック



大きな処理は区切る。



例:



// 移動処理

// 当たり判定

// アニメーション更新



\---



\# TODOコメント



一時対応には TODO を付ける。



例:



// TODO:

// 仮実装のため後で差し替え



\---



\# 関数コメント



複雑な関数のみ説明を書く。



例:



/\*\*



\* プレイヤー移動処理

\* 壁との衝突を考慮して移動する

&#x20; \*/



\---



\# コメント禁止事項



* コードを読めば分かる説明
* 古い仕様説明
* 長文日記
* コメントアウト放置



\---



\# 良い例



// 坂道で速度が増えすぎるのを防ぐ



\---



\# 悪い例



// speedを足す



\---



\# デバッグコード



不要になったデバッグコードは削除する。



コメントアウトで残さない。
